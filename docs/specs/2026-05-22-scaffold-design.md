# Scaffold Design: apps/web + pnpm Workspace

**Issue:** #2 — Scaffold apps/web: Next.js + Payload CMS v3 + Tailwind + pnpm workspaces
**Date:** 2026-05-22
**Status:** Approved

---

## Approach

Use `create-payload-app` to generate the initial Next.js + Payload CMS v3 project, then adjust it to match project conventions: pnpm workspace structure, strict TypeScript, Oxlint + Oxfmt in place of ESLint + Prettier, Valibot env validation, and Docker Compose for local Postgres.

`create-payload-app` is used because it produces a working Payload + Next.js baseline (routing, `withPayload()` wiring, admin route group, initial `payload.config.ts`) that we know exactly how to trim. The adjustment cost is lower than building from scratch.

---

## Scope

**Included:**
- pnpm workspace root (`apps/web` + `packages/domain`)
- Next.js 15 App Router + Payload CMS v3 (single project, single deployment)
- PostgreSQL via Payload's Drizzle adapter
- Tailwind CSS (generated by `create-payload-app`, confirmed working)
- shadcn/ui base install (`shadcn init` inside `apps/web` — sets up `components/ui/`, updates `tailwind.config.ts`; no components added yet)
- Oxlint + Oxfmt (replaces `create-payload-app`'s ESLint + Prettier)
- Vitest configured and working (`pnpm test` passes)
- Valibot env validation (`DevConfig | ProdConfig` discriminated union)
- Docker Compose for local Postgres
- Basic app router structure: `app/(frontend)/` and `app/(payload)/`

**Excluded:**
- Content collections (Jobs, Grants, Events, etc.) — separate issues
- Brand token application — tracked in `docs/specs/2026-05-22-brand-tokens-design.md`
- GitHub Actions CI — separate issue
- Storybook — part of the design system issue (MVP step 2)

## Section 2: TypeScript Configuration

A shared `tsconfig.base.json` at the repo root defines strict rules once. Both workspace packages extend it.

**`tsconfig.base.json`:**
```json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "noImplicitOverride": true,
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "lib": ["ES2022"],
    "skipLibCheck": true
  }
}
```

**`apps/web/tsconfig.json`** extends the base and adds Next.js-required options (`jsx`, `paths`, `plugins`). The `create-payload-app` generated tsconfig is replaced — its compiler options section is replaced with `extends: "../../tsconfig.base.json"` plus only the Next.js-specific additions.

**`packages/domain/tsconfig.json`** extends the base with `composite: true` and `declaration: true` for project references.

## Section 4: Payload CMS + Next.js Wiring

`create-payload-app` generates the core wiring. Targeted adjustments:

**`payload.config.ts`** — strip the example collection, keep a minimal valid config with only the `Users` collection (needed for Payload admin authentication):
```ts
export default buildConfig({
  admin: { user: 'users' },
  collections: [Users],
  db: postgresAdapter({ pool: { connectionString: env.DATABASE_URL } }),
  secret: env.PAYLOAD_SECRET,
})
```

Note: `env` is imported from `./env.ts` — no `process.env` access outside the validated env module.

**App router structure:**
- `app/(payload)/admin/[[...segments]]/` — generated by `create-payload-app`, kept as-is
- `app/(frontend)/page.tsx` — placeholder returning `<main>Work4Change</main>`, confirms routing works

## Section 5: Environment Variable Validation

**`apps/web/env.ts`** — imported at the top of `payload.config.ts` and `next.config.ts` before any other app code:

```ts
import * as v from 'valibot'

const DevConfig = v.object({
  NODE_ENV: v.literal('development'),
  DATABASE_URL: v.string(),
  PAYLOAD_SECRET: v.string(),
})

const ProdConfig = v.object({
  NODE_ENV: v.literal('production'),
  DATABASE_URL: v.pipe(v.string(), v.url()),
  PAYLOAD_SECRET: v.pipe(v.string(), v.minLength(32)),
})

export const env = v.parse(
  v.union([DevConfig, ProdConfig]),
  process.env,
)
```

All `process.env` access in `apps/web` goes through this module. Dev is intentionally lenient; prod enforces format and minimum secret length.
