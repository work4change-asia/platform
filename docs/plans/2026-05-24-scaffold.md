# Scaffold apps/web + pnpm Workspace Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap the monorepo with a working Next.js 15 + Payload CMS v3 dev environment using pnpm workspaces, strict TypeScript, Oxlint/Oxfmt, Valibot env validation, and Docker Compose for local Postgres.

**Architecture:** `create-payload-app` generates the `apps/web` baseline (Next.js + Payload wiring); we trim and adjust it to fit workspace conventions. A sibling `packages/domain` package is created as an empty shell. Tooling (Oxlint, Oxfmt, Vitest) is installed at the workspace root and replaces ESLint/Prettier.

**Tech Stack:** Next.js 15 App Router, Payload CMS v3, PostgreSQL (Payload Drizzle adapter), Tailwind CSS, shadcn/ui, Oxlint, Oxfmt, Vitest, Valibot, pnpm workspaces, Docker Compose

  "compilerOptions": {

    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "plugins": [{ "name": "next" }],

## Task 8: Valibot env validation module

**Files:**

- Create: `apps/web/env.ts`

- [ ] **Step 1: Install valibot in apps/web**

```bash
pnpm --filter web add valibot
```

- [ ] **Step 2: Create `apps/web/env.ts`**

```ts
import * as v from "valibot";

const DevConfig = v.object({
  NODE_ENV: v.literal("development"),
  DATABASE_URL: v.string(),
  PAYLOAD_SECRET: v.string(),
});

const ProdConfig = v.object({
  NODE_ENV: v.literal("production"),
  DATABASE_URL: v.pipe(v.string(), v.url()),
  PAYLOAD_SECRET: v.pipe(v.string(), v.minLength(32)),
});

export const env = v.parse(v.union([DevConfig, ProdConfig]), process.env);
```
