# Architecture Decision Records

Each file in this folder captures one significant decision: the context that forced it, what was chosen, what was rejected, and why. Decisions are numbered sequentially and never deleted — if a decision changes, the old record is marked `superseded` and a new one is added.

## Format

```
# ADR-NNNN: Title
Date: YYYY-MM-DD
Status: Accepted | Superseded by ADR-XXXX | Deprecated
```
